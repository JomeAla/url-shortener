API Documentation

Complete reference for the Shrtly URL Shortener REST API.

Authentication

Most endpoints require either a session cookie (logged-in via web) or an API key. API keys are passed via the X-API-Key header and can be created in your dashboard settings.

Session Auth: Login via POST /api/auth/login. The server sets a JSESSIONID cookie.

API Key Auth: Pass your key in the X-API-Key header on every request.

Public endpoints (no auth required) are marked Public.

URL Shortening

Create, list, update, and delete short URLs.
POST /api/shorten Public
Create a new short URL. Free-tier users get basic shortening; Pro/Enterprise users can use advanced options.
ParameterTypeRequiredDescription
urlstringYesThe long URL to shorten
customCodestringNoCustom short code (Pro+)
expiresAtISO datetimeNoLink expiration (Pro+)
passwordstringNoPassword protection (Pro+)
tagsstringNoComma-separated tags (Pro+)
utmSourcestringNoUTM source param (Pro+)
utmMediumstringNoUTM medium param (Pro+)
utmCampaignstringNoUTM campaign param (Pro+)
utmTermstringNoUTM term param (Pro+)
utmContentstringNoUTM content param (Pro+)
{ "url": "https://example.com/very/long/url", "customCode": "my-link", "tags": "news, blog" }
{ "shortCode": "abc123", "shortUrl": "http://localhost:8081/abc123", "longUrl": "https://example.com/very/long/url", "customCode": "my-link", "hasPassword": false, "expiresAt": null, "createdAt": "2026-05-28T00:00:00" }
POST /api/shorten/bulk Key/Session
Bulk create short URLs from an array of objects.
{ "urls": [ { "url": "https://example.com/1", "tags": "news" }, { "url": "https://example.com/2" } ] }
{ "results": [ { "url": "https://example.com/1", "shortUrl": "http://localhost:8081/abc", "success": true }, { "url": "https://example.com/2", "shortUrl": "http://localhost:8081/def", "success": true } ] }
GET /api/urls Key/Session
List all URLs for the authenticated user. Supports search via ?q=, ?dateFrom=, and ?dateTo= query parameters.
[ { "id": 1, "shortCode": "abc123", "customCode": "my-link", "longUrl": "https://example.com", "clickCount": 42, "hasPassword": false, "expiresAt": null, "createdAt": "2026-05-28T00:00:00" } ]
PUT /api/urls/{shortCode} Key/Session
Update a short URL's destination, custom code, expiry, password, tags, or UTM params.
{ "url": "https://new-destination.com", "customCode": "new-code", "tags": "updated, link" }
DELETE /api/urls/{shortCode} Key/Session
Delete a short URL permanently.

Analytics & Stats

Retrieve click data, time series, and export analytics.
GET /api/stats/{shortCode} Public
Get public click count for a short URL. Used internally for redirection.
GET /api/analytics/{shortCode} Key/Session
Full analytics for a short URL: total clicks, referrers, browsers, devices, recent clicks.
{ "totalClicks": 142, "referrers": { "twitter.com": 45, "direct": 80 }, "browsers": { "Chrome": 90, "Firefox": 30 }, "devices": { "Desktop": 100, "Mobile": 42 }, "recentClicks": [ { "timestamp": "2026-05-28T00:00:00", "referer": "twitter.com", "ip": "::1" } ] }
GET /api/analytics/{shortCode}/timeseries Key/Session
Click time series. Use ?interval=day (default) or ?interval=hour. Optional from and to ISO datetime params to filter the date range.
{ "interval": "day", "dataPoints": [ { "timestamp": "2026-05-27T00:00:00", "count": 12 }, { "timestamp": "2026-05-28T00:00:00", "count": 8 } ] }
GET /api/analytics/{shortCode}/export/csv Key/Session
Download click data as a CSV file.

Webhooks

Receive real-time event notifications via HTTP callbacks. Events include link.clicked and test. Payloads are signed with HMAC-SHA256 when a secret is configured.
POST /api/webhooks Key/Session
Create a new webhook endpoint. The events field accepts a comma-separated list of event types.
ParameterTypeRequiredDescription
urlstringYesHTTPS callback URL that receives webhook POST requests
eventsstringYesComma-separated event types (e.g. link.clicked)
secretstringNoHMAC-SHA256 signing secret (auto-generated if omitted)
{ "url": "https://myapp.com/webhooks/shrtly", "events": "link.clicked", "secret": "whsec_abc123..." }
{ "id": 1, "url": "https://myapp.com/webhooks/shrtly", "events": "link.clicked", "active": true, "createdAt": "2026-05-28T00:00:00" }
GET /api/webhooks Key/Session
List all webhooks for the authenticated user.
[ { "id": 1, "url": "https://myapp.com/webhooks/shrtly", "events": "link.clicked", "active": true, "createdAt": "2026-05-28T00:00:00" } ]
GET /api/webhooks/{id} Key/Session
Get a single webhook by ID. The secret field is masked.
{ "id": 1, "url": "https://myapp.com/webhooks/shrtly", "events": "link.clicked", "secret": "********", "active": true, "createdAt": "2026-05-28T00:00:00" }
PUT /api/webhooks/{id} Key/Session
Update a webhook's URL, events, secret, or active status. Only provided fields are updated.
{ "url": "https://myapp.com/webhooks/v2", "events": "link.clicked", "active": true }
DELETE /api/webhooks/{id} Key/Session
Delete a webhook permanently.
{"success": true}
POST /api/webhooks/{id}/test Key/Session
Dispatch a test event to the webhook. The test payload is fire-and-forget (dispatched asynchronously).
{"success": true, "message": "Test webhook dispatched"}
GET /api/webhooks/{id}/logs Key/Session
List delivery logs for a webhook, newest first.
[ { "id": 1, "event": "link.clicked", "success": true, "responseCode": 200, "errorMessage": null, "createdAt": "2026-05-28T00:00:00" } ]
GET /api/webhooks/{id}/logs/{logId} Key/Session
Get full detail for a single delivery log, including the request payload and response body.
{ "id": 1, "webhookId": 1, "event": "link.clicked", "payload": "{\"event\":\"link.clicked\",\"timestamp\":\"...\",\"data\":{...}}", "responseCode": 200, "responseBody": "{\"ok\":true}", "success": true, "errorMessage": null, "createdAt": "2026-05-28T00:00:00" }

Integrations

Connect Shrtly with n8n, Make (Integromat), and Zapier for automated workflows.

→ n8n

Trigger: Receive link clicks in n8n

  1. In n8n, add a Webhook node and copy the production URL (e.g. https://your-n8n.example.com/webhook/xyz).
  2. In Shrtly, go to Dashboard → Webhooks and create a new webhook pointing to that URL with event link.clicked.
  3. When someone clicks one of your short links, Shrtly sends an HTTP POST to your n8n webhook with the click data.

Action: Shorten a URL from n8n

  1. Add an HTTP Request node in n8n.
  2. Method: POST — URL: {{baseUrl}}/api/shorten
  3. Headers: add X-API-Key with your Shrtly API key (create one in Dashboard → API Keys).
  4. Body (JSON): {"url": "{{your_long_url}}"}
  5. The response contains shortUrl and shortCode for use in downstream nodes.

→ Make (Integromat)

Trigger: Receive link clicks in Make

  1. In Make, add a Webhook module and choose "Custom Webhook". Copy the webhook URL.
  2. In Shrtly, go to Dashboard → Webhooks and create a new webhook pointing to that URL with event link.clicked.
  3. Each click on your short links triggers the Make scenario.

Action: Shorten a URL from Make

  1. Add an HTTP module → "Make a request".
  2. Method: POST — URL: {{baseUrl}}/api/shorten
  3. Headers: X-API-Key with your Shrtly API key.
  4. Body type: JSON — {"url": "{{long_url}}"}

→ Zapier

Trigger: New link click

  1. In Shrtly, go to Dashboard → Webhooks and create a webhook with event link.clicked.
  2. In Zapier, add a Webhooks by Zapier trigger → "Catch Hook".
  3. Copy the Zapier webhook URL into Shrtly's webhook URL field and save.
  4. Zapier will receive click events in real time.

Action: Create short URL

  1. In Zapier, add a Webhooks by Zapier action → "POST".
  2. URL: {{baseUrl}}/api/shorten
  3. Headers: X-API-Key with your Shrtly API key.
  4. Payload type: JSON — {"url": "{{long_url}}"}

All three platforms support Bulk Shorten via POST /api/shorten/bulk and Analytics via GET /api/analytics/{code} using the same API key authentication pattern.
